Tool Reference

Every tool available to the AI, organized by category. Tools can be individually enabled/disabled with tool_manage.

Context Management

context_close

Closes dynamic context panels by their IDs. Cannot close fixed panels (P0–P10).

ParameterTypeRequiredDescription
idsstring[]✓Panel IDs to close (e.g., ["P12", "P15"])

log_create

Creates timestamped log entries for recording decisions, actions, or events.

ParameterTypeRequiredDescription
entriesobject[]✓Array of {content: string}

log_summarize

Summarizes multiple top-level log entries into a single parent summary. Original logs become children hidden under the summary. Minimum 10 entries required.

ParameterTypeRequiredDescription
log_idsstring[]✓Log IDs to summarize (min 10, must be top-level)
contentstring✓Summary text for the new parent log

log_toggle

Expands or collapses a log summary to show or hide its children.

ParameterTypeRequiredDescription
idstring✓Log ID (must be a summary with children)
actionstring✓expand or collapse

close_conversation_history

Closes a ConversationHistory panel, optionally creating logs and memories to preserve important information.

ParameterTypeRequiredDescription
idstring✓Panel ID of the history panel
logsobject[]Log entries to create
memoriesobject[]Memory items to create

ask_user_question

Presents 1-4 multiple-choice questions to the user as an interactive form. Use this to gather user preferences, clarify ambiguous instructions, or get implementation decisions from the user.

The form replaces the input field until the user submits answers or presses Esc to dismiss. Each question supports 2-4 options, with an auto-appended "Other" option for free-text input. Questions can allow single or multiple selections.

ParameterTypeRequiredDescription
questionsobject[]✓Array of 1-4 question objects

Question object structure:

  • question (string, required) — The question text
  • options (string[], required) — 2-4 answer options. If recommending an option, list it first with "(Recommended)" in the label
  • multiSelect (boolean, optional) — Allow multiple selections (default: false)
Example Use Cases
  • Ask which authentication method to implement (OAuth, JWT, Session-based)
  • Clarify which database to use (PostgreSQL, MySQL, SQLite)
  • Get preference on code style or framework choices
  • Confirm destructive operations before proceeding

File Operations

file_open

Opens a file and adds it to context. Always use this before file_edit.

ParameterTypeRequiredDescription
pathstring✓Path to the file

file_edit

Edits a file by replacing exact text. Preferred over file_write for modifications.

ParameterTypeRequiredDescription
file_pathstring✓Absolute path to file
old_stringstring✓Exact text to find and replace
new_stringstring✓Replacement text
replace_allbooleanReplace all occurrences (default: false)

file_write

Creates a new file or completely replaces all content. Use file_edit for targeted changes.

ParameterTypeRequiredDescription
file_pathstring✓Path to the file
contentsstring✓Complete file contents

file_glob

Searches for files matching a glob pattern. Creates an auto-refreshing panel.

ParameterTypeRequiredDescription
patternstring✓Glob pattern (e.g., **/*.rs)
pathstringBase path to search from

file_grep

Searches file contents for a regex pattern. Creates an auto-refreshing panel.

ParameterTypeRequiredDescription
patternstring✓Regex pattern
pathstringBase path to search from
file_patternstringGlob filter for files (e.g., *.rs)

Tree

tree_filter

Edits the gitignore-style filter for the directory tree view.

ParameterTypeRequiredDescription
filterstring✓Gitignore-style patterns, one per line

tree_toggle

Opens or closes folders in the directory tree.

ParameterTypeRequiredDescription
pathsstring[]✓Folder paths to toggle
actionstringopen, close, or toggle

tree_describe

Adds or updates descriptions for files and folders. Descriptions appear in the tree and are marked stale when files change.

ParameterTypeRequiredDescription
descriptionsobject[]✓Array of {path, description, delete?}

Git

git_execute

Executes a git command. Read-only commands create auto-refreshing panels; mutating commands run directly.

ParameterTypeRequiredDescription
commandstring✓Full git command (e.g., git log --oneline -10)

git_configure_p6

Configures the P6 git status panel.

ParameterTypeRequiredDescription
show_diffsbooleanShow full diff content
show_logsbooleanShow recent commit history
log_argsstringCustom git log arguments
diff_basestringGit ref to diff against (empty to clear)

GitHub

gh_execute

Executes a GitHub CLI command. Requires GITHUB_TOKEN.

ParameterTypeRequiredDescription
commandstring✓Full gh command (e.g., gh pr list)

Console (tmux)

console_create

Creates a console context panel to monitor terminal output.

ParameterTypeRequiredDescription
pane_idstringExisting tmux pane ID (omit to auto-create)
descriptionstringDescription of what this console is for
linesintegerNumber of lines to capture

console_edit

Updates an existing console's configuration.

ParameterTypeRequiredDescription
context_idstring✓Console panel ID (e.g., P7)
descriptionstringNew description
linesintegerNumber of lines to capture

console_send_keys

Sends keystrokes to a console pane. Enter is sent automatically after the keys.

ParameterTypeRequiredDescription
context_idstring✓Console panel ID
keysstring✓Keys to send (e.g., ls -la or C-c)

console_sleep

Non-blocking 1-second pause. UI remains responsive during the wait.

Todo

todo_create

Creates one or more todo items. Supports nesting via parent_id.

ParameterTypeRequiredDescription
todosobject[]✓Array of {name, description?, parent_id?}

todo_update

Updates existing todos: status, name, description, parent, or delete.

ParameterTypeRequiredDescription
updatesobject[]✓Array of {id, status?, name?, description?, parent_id?, delete?}

todo_move

Moves a todo to a new position in the list.

ParameterTypeRequiredDescription
idstring✓Todo ID to move
after_idstringPlace after this ID (null = move to top)

Memory

memory_create

Creates persistent memory items.

ParameterTypeRequiredDescription
memoriesobject[]✓Array of {content, contents?, importance?, labels?}

memory_update

Updates or deletes existing memory items.

ParameterTypeRequiredDescription
updatesobject[]✓Array of {id, content?, contents?, importance?, labels?, open?, delete?}

Scratchpad

scratchpad_create_cell

Creates a new scratchpad cell.

ParameterTypeRequiredDescription
cell_titlestring✓Cell title
cell_contentsstring✓Cell content

scratchpad_edit_cell

Edits an existing scratchpad cell.

ParameterTypeRequiredDescription
cell_idstring✓Cell ID (e.g., C1)
cell_titlestringNew title
cell_contentsstringNew content

scratchpad_wipe

Deletes scratchpad cells. Empty array deletes all.

ParameterTypeRequiredDescription
cell_idsstring[]✓Cell IDs to delete (empty = all)

Command tools

ToolDescription

Spine

notification_mark_processed

Marks a notification as processed.

ParameterTypeRequiredDescription
idstring✓Notification ID (e.g., N1)

spine_configure

Configures auto-continuation and guard rails. All parameters optional.

ParameterTypeDescription
max_tokens_auto_continuebooleanAuto-continue on truncation (default: true)
continue_until_todos_donebooleanKeep going until all todos done (default: false)
max_auto_retriesinteger?Max consecutive auto-continuations (null to disable)
max_costnumber?Max cost in USD (null to disable)
max_duration_secsinteger?Max autonomous duration in seconds
max_messagesinteger?Max conversation messages
max_output_tokensinteger?Max total output tokens
reset_countersbooleanReset runtime counters

System

system_reload

Reloads the TUI application. State is preserved. Use after modifying source code and rebuilding.

tool_manage

Enables or disables tools. The tool_manage tool itself cannot be disabled.

ParameterTypeRequiredDescription
changesobject[]✓Array of {tool, action} where action is "enable" or "disable"

module_toggle

Activates or deactivates modules. Core modules cannot be deactivated.

ParameterTypeRequiredDescription
changesobject[]✓Array of {module, action} where action is "activate" or "deactivate"

preset_snapshot_myself

Saves current configuration as a named preset.

ParameterTypeRequiredDescription
namestring✓Preset name (alphanumeric + hyphens)
descriptionstring✓What this preset is for
replacestringName of existing preset to overwrite

preset_load

Loads a named preset, replacing current configuration.

ParameterTypeRequiredDescription
namestring✓Preset name to load
