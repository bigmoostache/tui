Context Pilot Documentation

Everything you need to master Context Pilot — from installation to autonomous multi-hour coding sessions.

What is Context Pilot?

Context Pilot is an AI-native terminal IDE built in Rust. It gives you full, transparent control over what your AI assistant sees — every file, diff, search result, and note is a context panel with a visible token count.

Unlike tools that hide context management behind abstractions, Context Pilot makes it the primary interface. You see exactly what the AI sees. You decide what stays and what goes.

Key Insight

The quality of AI output is directly proportional to the quality of its context. Context Pilot gives you the instruments to tune that context precisely.

Installation

Prerequisites

  • Rust toolchain — install via rustup.rs
  • tmux — required for console integration (apt install tmux or brew install tmux)
  • An API key from a supported LLM provider (Anthropic, OpenAI, DeepSeek, Grok, Groq)

Build from source

git clone https://github.com/bigmoostache/context-pilot.git
cd context-pilot
cargo build --release

The binary will be at target/release/tui. You can copy it to your PATH or create an alias.

Configure your API key

# Create .env in the directory where you'll run Context Pilot
echo 'ANTHROPIC_API_KEY=sk-ant-...' > .env

# Or set as environment variable
export ANTHROPIC_API_KEY=sk-ant-...
Multiple Providers

You can configure multiple providers simultaneously. See Configuration for all supported environment variables.

Quickstart

Launch

# Navigate to your project
cd your-project

# Launch inside tmux (required for console features)
tmux new-session context-pilot

# Or if already in tmux
context-pilot

Your first interaction

When Context Pilot starts, you'll see the sidebar on the left listing context panels (P0–P10 are built-in), and the main conversation area on the right. Just start typing and press Enter to send.

The AI can see everything listed in the sidebar. It can open files, run commands, search code — all through tools that create new context panels.

Essential workflow

  1. Ask the AI to explore — it will open files, grep for patterns, check git status
  2. Watch context grow — new panels appear in the sidebar with token counts
  3. Close what you don't need — select a panel and press d to close it
  4. Stay under budget — the token bar shows usage against your budget

Core Concepts

Context Panels

Everything the AI sees is a panel. Panels have IDs (P0, P1, P7, P12…), a type, and a token count. Fixed panels (P0–P10) are always present. Dynamic panels are created by tools and can be closed.

PanelTypeDescription
P0SystemActive agent (system prompt)
P1ChatCurrent conversation
P2TreeDirectory structure
P3TodosTask tracking
P4MemoriesPersistent notes
P5OverviewContext usage & stats
P6GitBranch, status, diffs
P7ScratchpadTemporary cells
P8LibraryAgents, skills, commands
P9SpineNotifications & auto-continuation
P10LogsTimestamped event log with tree-structured summarization

Modules

Modules are independent packages of functionality. Each module provides its own tools (things the AI can do) and panels (things the AI can see). You can activate or deactivate modules to control what's available. See Modules for the full list.

Tools

Tools are actions the AI can take — opening files, running git commands, creating todos. Tools are grouped by module and can be individually enabled or disabled. See Tool Reference for complete documentation.

Agents & Skills

Agents are system prompts that define the AI's behavior. Skills are knowledge documents loaded as context panels. Commands are text shortcuts expanded inline. See Agents & Skills.

Presets

Presets snapshot your entire configuration — active agent, modules, disabled tools, panel state — into a named profile you can load instantly. Built-in presets include Worker, Planner, Context Builder, and Context Cleaner.
